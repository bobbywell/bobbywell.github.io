网站提速方法

发表于 2022-04-25 更新于 2022-05-06 分类于 网站建设

网站断断续续搞了4个月，到今天基本收工。

下一步就是提升网站访问速度。


01 先测试一下网站优化前的速度

测试网站的工具网站很多，直接Google搜索“website speed test”，主要用的有几个：

https://gtmetrix.com/

https://pagespeed.web.dev/

https://tools.pingdom.com/


  • 不要盲目相信测试分数，一切以网站实际体验为依据；即使分数低只要浏览体验没问题也无须过度优化；
  • 不同工具测试分数可能差别很大；



02 网站加载速度慢的原因

从以上的测试来看速度很慢，测试工具也给出了一些原因，主要有以下：

（1）服务器响应慢；

这个是可以预料到的，换主机太麻烦了，考虑用DNS解决；

（2）首页图片体积太大；

转换格式，将PNG转成WEB；

（3）图片没有设置延迟加载；

可以使用lazy load功能的插件；



03 网站速度优化项

速度优化可以从下面几个步骤开始：

有一个工具可以很好的判断网站是否有启用gzip压缩功能，https://www.giftofspeed.com/gzip-test/

可以看到，cloudfare为网站提供了压缩功能，虽然用的主机不是它，真的良心。免费都能这么好。



04 加载速度优化步骤

优化之前，还是先决定对整个网站做个备份。

这一步挺重要的。

网站出现意外情况，随时恢复到优化前。


首先，考虑到还没有安装优化插件，所以第一步先打算用插件把能优化的都做了，因为插件集成了很多功能，没必要自己一项一项去单独优化，最后造成插件冗余。

加速优化插件的选择很多，常见的有LiteSpeed Cache, Autoptimize, WP Rocket, Rank math等等。在shoptimizer的说明文档中，它推荐的是autoptimize，所以就用这个好了。

autoptimize设置按照shoptimizer的文档中建议就好：https://www.commercegurus.com/docs/shoptimizer-theme/speed-tips/#

看起来还是有点效果。


由于服务器和目标市场不在一个国家，所以服务器在当地的响应速度肯定不是最快的，我能想到的就是用CDN缓存。目前来说，免费的又最好用的莫过于cloudfare.

couldfare的注册和使用是很简单的。具体可以参考以下：

开通CloudFlare CDN加速教程

How to set up cloudfare for wordpress

How to Set Up Cloudflare CDN for WordPress Blog

很遗憾，cloudfare设置好以后，网站页面加载不正常，图片不显示或者排版错乱，完全无法浏览。

什么原因呢？


自建站就是这样，你永远不知道哪一步会出问题。

  • 先是在网上找到了类似的解决方案，安装FlexibleSSL for cloudfare插件，没有效果；

  • 然后将cloudfare设置成开发者模式，绕过缓存，仍然没有效果；

  • 按照故障排除指导暂停cloudfare，暂停roket load，没有效果；


没办法，只能祭出恢复备份的终极大招了。

这就是自建站需要随时备份的重要性了。


恢复备份，账号正常了。

但是之前的优化也归零。
